# دکتر (عیب‌یاب راهنما)

دستور `mhrv-f doctor` یک عیب‌یاب «اولین اجرا» است که رایج‌ترین مشکل‌ها را بررسی می‌کند و راه‌حل‌های عملی پیشنهاد می‌دهد.

هر وقت این‌ها را دیدید، اجرا کنید:

- `504 Relay timeout`
- خطاهای گواهی مثل `NET::ERR_CERT_AUTHORITY_INVALID`
- «دیروز کار می‌کرد، امروز تایم‌اوت می‌شود»
- وقتی مطمئن نیستید مسیر رله واقعاً بالا هست یا نه

## اجرا

```bash
./mhrv-f doctor
```

یا در UI دسکتاپ:

- روی **Doctor** (کنار **Test relay**) بزنید و خروجی را در بخش **Recent log** ببینید.

## چه چیزهایی را چک می‌کند؟

- **هشدارهای تنظیمات**: خطر `LAN`، ضعیف بودن `auth_key`، `verify_ssl=false` و …
- **درستی حالت**: `apps_script` / `direct` / `full`
- **پول‌های Apps Script**: وجود حداقل یک `account_groups` فعال (وقتی لازم است)
- **گواهی MITM** (در حالت‌های غیر `full`): فایل CA ساخته شده و قابل اعتماد بودن آن
- **تست کامل رله**: همان تست `mhrv-f test` را اجرا می‌کند و خطاهای رایج را طبقه‌بندی می‌کند

## راه‌حل‌های رایج

- **CA قابل اعتماد نیست**: `mhrv-f --install-cert` را با دسترسی ادمین اجرا کنید یا `ca/ca.crt` را در Trust Store سیستم اضافه کنید.
- **تست رله Fail می‌شود**:
  - مطمئن شوید `AUTH_KEY` در `Code.gs` دقیقاً با `auth_key` در `config.json` یکی است
  - اگر Deployment مرده: در Apps Script یک “New version” Deploy کنید و ID جدید را جایگزین کنید
  - `google_ip` را عوض کنید / SNI pool را تست کنید
  - اگر سهمیه تمام شده: حساب/Deployment پشتیبان اضافه کنید
