# امنیت و اعتماد (به زبان ساده)

## این ابزار چه می‌کند (و چرا CA لازم است؟)

در حالت Apps Script (MITM)، **MasterHttpRelayVPN-Frankestein** اتصال HTTPS را **روی دستگاه شما** باز می‌کند، بایت‌ها را از طریق Apps Script شما رله می‌کند، سپس دوباره به سمت مقصد واقعی رمز می‌کند.

به همین دلیل نیاز دارد یک **مرجع گواهی محلی (CA)** روی دستگاه شما مورد اعتماد باشد تا مرورگر/اپ‌ها گواهی‌های ساخته‌شدهٔ محلی را بپذیرند.

## چه چیزی کجا رمزگشایی می‌شود؟

- **روی دستگاه شما رمزگشایی می‌شود**: در MITM، متن خام HTTPS برای پردازهٔ محلی قابل مشاهده است (مثل هر پروکسی دیباگ).
- **برای ISP شما رمزگشایی نمی‌شود**: ISP فقط یک اتصال TLS بیرونی می‌بیند که شبیه ترافیک عادی گوگل است.

## فایل‌های CA شما

در پوشهٔ تنظیمات، این فایل‌ها را می‌بینید:

- `ca/ca.crt` — گواهی عمومی (فقط اگر دقیقاً می‌دانید چه می‌کنید به اشتراک بگذارید)
- `ca/ca.key` — کلید خصوصی (**هرگز به اشتراک نگذارید، commit نکنید، آپلود نکنید**)

اگر کسی `ca.key` را داشته باشد و دستگاه‌های شما به آن CA اعتماد کنند، می‌تواند برای شما سایت‌ها را جعل کند.

## نصب / تعمیر / حذف

- **نصب**: از launcher (`run.bat` / `run.command` / `run.sh`) یا `mhrv-f --install-cert` استفاده کنید.
- **تعمیر**: اگر خطای گواهی دیدید، CA را دوباره نصب/اعتماد کنید و سپس Doctor را اجرا کنید.
- **حذف (uninstall)**:
  - CA را از trust store سیستم‌عامل حذف کنید
  - پوشهٔ `ca/` را از پوشهٔ تنظیمات برنامه پاک کنید

## نکتهٔ اندروید (Android 7+)

در اندروید، هر اپ می‌تواند تصمیم بگیرد به CAهای نصب‌شده توسط کاربر اعتماد نکند. مرورگرها معمولاً اعتماد می‌کنند؛ بانک/پیام‌رسان‌ها اغلب نه. این رفتار طبیعی اندروید است. برای آن اپ‌ها از proxy-only یا split tunneling استفاده کنید.
