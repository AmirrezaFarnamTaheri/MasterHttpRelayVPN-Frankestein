# عیب‌یابی (علت → راه‌حل)

## «Connect/Start غیرفعال است» (اندروید) یا «To start, fix: …» (دسکتاپ)

- **علت محتمل**: در حالت Apps Script، اطلاعات رله (Deployment URL/ID و `auth_key`) کامل نیست.
  - **راه‌حل**: حداقل یک `account_groups` فعال با `script_ids` و `auth_key` درست اضافه کنید.
- **علت محتمل**: پورت‌ها نامعتبر یا تداخل دارند.
  - **راه‌حل**: پورت‌ها عددی باشند؛ HTTP و SOCKS5 یکی نباشند.

## «504 Relay timeout»

- **علت محتمل**: دیپلوی Apps Script شما مشکل دارد (قدیمی/مسدود/به سقف quota خورده).
  - **راه‌حل**:
    - در Apps Script یک “New version” deploy کنید و Deployment ID جدید را بگذارید
    - چند Deployment ID اضافه کنید تا بار پخش شود
    - یک Google account group دیگر اضافه کنید تا quota بیشتر شود

## «یوتیوب باز می‌شود ولی ویدیو بافر/تکه‌تکه است»

- **علت محتمل**: بایت‌های ویدیو از `*.googlevideo.com` می‌آید و ممکن است تعداد زیادی فراخوانی Apps Script ایجاد کند (فشار call-count + quota)، مخصوصاً روی کیفیت‌های بالا.
  - **راه‌حل**:
    - Deployment IDهای بیشتر / account groupهای بیشتر اضافه کنید (ظرفیت)
    - fan-out را کمتر کنید (`parallel_relay`، `range_parallelism`) یا `relay_rate_limit_qps` را فعال کنید تا جهش‌ها صاف شود
    - نکته: برای URLهای `googlevideo.com` که `clen=` دارند، موتور به‌صورت خودکار چانک‌ها را بزرگ‌تر و همزمانی را محدودتر می‌کند تا تعداد فراخوانی‌های Apps Script کمتر شود

## خطای گواهی (مثل `NET::ERR_CERT_AUTHORITY_INVALID`)

- **علت محتمل**: CA محلی MITM روی این دستگاه مورد اعتماد نیست (یا در اندروید ۷+ اپ موردنظر به CA کاربر اعتماد نمی‌کند).
  - **راه‌حل**:
    - CA را دوباره نصب/اعتماد کنید
    - در اندروید: بعضی اپ‌ها هرگز به CA کاربر اعتماد نمی‌کنند؛ از proxy-only یا split tunneling استفاده کنید
- **نکتهٔ امنیتی**: فایل `ca.key` را هرگز به اشتراک نگذارید.

## «می‌گوید وصل شد، ولی سایت‌ها لود نمی‌شوند»

- **علت محتمل**: `google_ip` اشتباه است یا `front_domain`/SNI pool مشکل دارد.
  - **راه‌حل**:
    - تست SNI (`mhrv-f test-sni` یا تستر داخل UI)
    - اسکن IP (`mhrv-f scan-ips`)
    - یک `google_ip` شناخته‌شدهٔ دیگر را امتحان کنید
- **علت محتمل**: `auth_key` اشتباه است.
  - **راه‌حل**: `AUTH_KEY` را از `Code.gs` دقیق کپی کنید (مراقب فاصلهٔ آخر رشته).

## «UDP در Full Tunnel کار نمی‌کند»

- **علت محتمل**: ناسازگاری نسخه‌ها
  - **راه‌حل**:
    - اگر tunnel-node قدیمی باشد ممکن است UDP را `UNSUPPORTED_OP` برگرداند و کلاینت به TCP-only برگردد
    - tunnel-node و کلاینت را به نسخه‌های سازگار ارتقا دهید

## «script.google.com بلاک است (نمی‌توانم رله را deploy کنم)»

- **راه‌حل**: از حالت **Direct (bootstrap)** استفاده کنید تا `script.google.com` باز شود، deploy را انجام دهید، سپس به Apps Script برگردید.
