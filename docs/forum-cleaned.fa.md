# راهنمای تکمیلی (فارسی) — MasterHttpRelayVPN-Frankestein

> نسخه‌ی تمیز و نگهداری‌شده‌ی نکات انجمن و چتِ خام است؛ فایل خام آرشیو دیگر همراه ریپو نگه داشته نمی‌شود.

راهنمای زیر نکات تکمیلی برای استفاده از کلاینت **MasterHttpRelayVPN-Frankestein** (`mhrv-f`) و رله‌ی Google Apps Script است.

## 1) مهم‌ترین واقعیت میدانی

- **تمام‌ شدن quota گوگل (UrlFetchApp)** شایع‌ترین علت «اول کار می‌کند، بعد از چند ساعت/روز تایم‌اوت می‌دهد» است.
- برای پایداری واقعی **چند Deployment ID** و ترجیحاً **چند اکانت گوگل** داشته باشید.

## 2) خطاهای پرتکرار و راه‌حل سریع

### A) `bad json` / خطاهای Deploy

- اگر هنگام Deploy به خطاهای عجیب (مثلاً اشاره به یک خط خاص) برخوردید:
  - متن `Code.gs` را **کامل انتخاب کنید → کات/پاک کنید → دوباره پیست کنید** و دوباره Deploy کنید.
  - بعد از Deploy، قبل از کپی Deployment ID، روی لینک اجرای deployment (اجرای اولیه) کلیک کنید تا یک بار execute شود.

### B) CA / نصب Certificate

- اگر HTTPS باز نمی‌شود یا مرورگر هشدار می‌دهد:
  - از `run.bat` یا `mhrv-f --install-cert` استفاده کنید.
  - در فایرفاکس ممکن است لازم باشد `ca/ca.crt` را دستی Import کنید.

### C) «شناسایی به عنوان ربات»

- هر درخواستی که از Apps Script relay عبور کند با `User-Agent: Google-Apps-Script` شناخته می‌شود و بعضی سایت‌ها نسخه‌ی ساده/کپچا می‌دهند.
- برای دامنه‌هایی که مشکل دارند:
  - اگر دامنه گوگل‌ـ‌اج است، از مسیر **SNI rewrite** استفاده کنید (در `hosts` یا `domain_overrides`).
  - اگر با MITM مشکل دارد، آن دامنه را **مستقیم** کنید:

```json
{
  "domain_overrides": [
    { "host": "problem.example", "force_route": "direct", "never_chunk": true }
  ]
}
```

### D) کندی ویدیو (YouTube)

- صفحه‌ی یوتیوب ممکن است سریع باز شود، اما chunkهای ویدیو quota می‌خورند.
- راه‌حل‌های عملی: چند `script_id`، چند `account_groups`، و در صورت نیاز `relay_rate_limit_qps`.
- نکتهٔ فنی: کلاینت برای دانلودهای بزرگ از **range-parallel** استفاده می‌کند (probe با `Range` و سپس چند chunk موازی). برای یوتیوب/گوگل‌ویدیو اگر URL شامل `clen=` باشد، برنامه چانک‌ها را بزرگ‌تر می‌کند و همزمانی را محدود می‌کند تا تعداد فراخوانی‌های Apps Script کمتر و فشار quota کمتر شود.

### E) WebSocket کار نمی‌کند

- WebSocket از ذات روش Apps Script پشتیبانی نمی‌کند (streaming real-time و موارد مشابه).

## 3) تنظیم پروکسی (خلاصه)

- HTTP: `127.0.0.1:8085` — SOCKS5: `127.0.0.1:8086`

## 4) چک‌لیست قبل از گزارش مشکل

1) `mhrv-f doctor` یا `mhrv-f doctor-fix`  
2) `mhrv-f support-bundle`  
3) اگر پس از مدتی شکست شروع شد، **quota** و **چند deployment/اکانت** را جدی بگیرید.
